## Core Principles

Every implementation and task execution within this framework must uphold:

1.  **Reality before assumption.**
2.  **Evidence before conclusion.**
3.  **Truth before convenience.**
4.  **Humans remain accountable.**
5.  **Respect dignity, privacy, and consent.**
6.  **Explain significant reasoning where appropriate.**
7.  **Preserve provenance and history.**
8.  **Design for interoperability.**
9.  **Learn continuously.**
10. **Steward for future generations.**

## Universal Architecture

The repository is organized according to the nine layers of the OceanicOS Universal Architecture:

*   **Layer 0: Reality** - Observation, Evidence, Measurement, Context.
*   **Layer 1: Constitution** - Mission, Principles, Governance, Standards.
*   **Layer 2: Human Kernel** - Identity, Knowledge, Memory, Reasoning, Learning.
*   **Layer 3: Core Engines** - Reality, Memory, Knowledge, Verification, Decision, Planning, Simulation, Builder, Registry, Security.
*   **Layer 4: Intelligence** - Search, Knowledge Graph, Reasoning, Recommendations, Evaluation, Planning, Learning.
*   **Layer 5: Builder** - CLI, API, SDK, Automation, Documentation, Testing, Versioning.
*   **Layer 6: Applications** - Web, Mobile, Desktop, Terminal, Embedded Systems, Extensions.
*   **Layer 7: Collaboration** - Individuals, Teams, Organizations, Communities, Governance, Shared Knowledge.
*   **Layer 8: Infrastructure** - Local, Cloud, Offline, Synchronization, Containers, Deployment, Monitoring.
*   **Layer 9: Stewardship** - Security, Privacy, Accessibility, Maintenance, Migration, Long-term Preservation, Continuous Improvement.
