Thank you for helping shape OceanicOS. This project is intentionally lightweight and open to new ideas.

## How to contribute

- Open an issue if you want to discuss a new idea, proposal, or change.
- Open a pull request for direct improvements to the charter, roadmap, architecture, governance, or documentation.
- Keep contributions focused on clarity, accessibility, transparency, and alignment with the charter values.

## Good first tasks

- Propose practical deliverables for OceanicOS.
- Help define the first audience and use cases.
- Suggest a simple decision-making process for charter updates.
- Improve the wording or structure of the charter so it is easier to read.
- Help refine the planning, memory, or tool integration model.

## Review process

- Simple documentation changes can move quickly.
- Substantive updates should include a short explanation of why the change matters.
- Major changes should be linked to a decision record or proposal.
- Revisions should preserve the living, open, agnostic, and human-centered character of the project.

## Keep it alive

- Revisit the roadmap regularly.
- Update the charter when the project learns something new.
- Encourage others to contribute by keeping the process welcoming and transparent.
