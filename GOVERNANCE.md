# Governance

OceanicOS uses a lightweight stewardship model so the project stays open, adaptive, and accountable without becoming overly bureaucratic.

## Roles

- Contributors: propose ideas, documentation edits, and implementation work.
- Maintainers: review changes, preserve alignment with the constitution, and guide the project direction.
- Stewards: coordinate long-term health, roadmap priorities, and public communication.

## Decision Process

1. Proposals should be documented in an issue, pull request, or decision record.
2. Changes should be discussed openly and reviewed for alignment with the constitution.
3. Significant changes should be captured in the decision record system.
4. Maintainers may merge routine updates; major changes should receive broader review.

## Amendment Process

Any change to the charter, constitution, architecture, or memory model should be proposed transparently and reviewed before adoption. The project should favor reversible changes and clear rationale.

## Review Cadence

The project should revisit its core documents periodically to ensure they remain useful and current.
